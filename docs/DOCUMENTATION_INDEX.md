# Purpose

This document is the master index of the SmartFlow documentation.

It provides a high-level overview of the documentation structure and serves as the primary entry point for developers, architects, AI assistants, and reviewers.

Project-specific documentation remains inside its canonical location.

# Documentation Principles

- Every document has a single canonical location.
- Documents should not be duplicated.
- Major architectural changes require an ADR.
- Standards take precedence over informal notes.
- PROJECT_STATUS.md represents the current implementation status.
- Documentation should evolve together with the codebase.
